Example Coding Exercise

Check if the input string is a Palindrome

A string is a palindrome if it reads the same forwards and backwards.

Given a string "s", return true if it is a palindrome, or false otherwise.

Example 01:

Input: s = "car"

Output: false

Explanation: "car" is not a palindrome

Example 02:

Input: s = "kayak"

Output: true

Explanation: "kayak" is a palindrome

Constraints

  • 1 <= s.length <= 2 * 10^5
  • s consists only of printable ASCII characters

Example Solution Overview

Our detailed code walkthrough videos set us apart from competing products. We have designed them to mimick an ideal answer for a coding interview, consisting of discussing a brute force approach followed by an optimized approach.

Brute Force Approach - Reverse and Compare

Summary: To check if a string is a palindrome or not we can reverse the string and compare it with the original string. If the reversed string is equal to the original string then it is a palindrome.

Time Complexity: O(n), where n is the length of the given string. However, it requires multiple full traversals of the string since splitting the string into an array takes O(n), reversing the array also takes O(n) time, and joining the array also takes O(n) time. In total all these operations take O(n) + O(n) + O(n).

Optimized Approach - Two Pointers

Summary: We can use two pointers to achieve this in one traversal through the characters in the string avoiding any extra memory. Create two pointers, one at the start and one at the end of the string, then move the start and end pointers towards the middle index. If the next set of characters are equal the string is still a valid palindrome.

Time Complexity: O(n), where n is the length of the given string. However, this solution only does one traversal through the string.
